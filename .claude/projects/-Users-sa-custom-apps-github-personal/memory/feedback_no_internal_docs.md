---
name: Don't expose internal architecture in public profile
description: Never pull from internal architecture docs (modules.md, interface-contracts.md, etc.) into the public GitHub profile README
type: feedback
---

Don't surface internal architecture details (dependency layers, module structure, interface contracts) in the public GitHub profile. Those docs are for the codebase, not for the world.

**Why:** User considers architecture docs internal — exposing them on a public profile feels like leaking implementation details, not showcasing work.

**How to apply:** When editing the GitHub profile README, keep content at the product/vision level. Never reference or condense content from architecture docs, dev-skills, or product-stories into the profile.
